# Use Case 3.11.0: Manage Article

## 1. Use Case Description

| Field              | Content                                                                                                                                  |
| :----------------- | :--------------------------------------------------------------------------------------------------------------------------------------- |
| **Name**           | Manage Article                                                                                                                           |
| **Description**    | This use case allows the **Admin** to choose one of the corresponding functions of CRUD to adjust **Article** information in the system. |
| **Actor**          | **Admin**, **All (Public for viewing)**                                                                                                  |
| **Trigger**        | When the **Admin** clicks on the "**Article**" button on the navigation sidebar.                                                         |
| **Pre-condition**  | 1. **Admin**'s device must be connected to the internet.<br>2. **Admin** is signed in with their account.                                |
| **Post-condition** | The **Article** information is updated to the corresponding function that the **Admin** executes.                                        |

---

## 2. Sequence Flow

```plantuml
@startuml
autonumber
skinparam sequenceMessageAlign center

actor "Admin" as User
boundary "ArticleManagementPage" as View
control "ArticleController" as Controller
entity "ArticleRepository" as Repo

User -> View: Access Article Management
View -> Controller: GET /articles
activate Controller
Controller -> Repo: findAll()
activate Repo
Repo --> Controller: ArticleList
deactivate Repo
Controller --> View: Return list data
deactivate Controller
View --> User: Display list of Articles

User -> View: Choose function

opt List All Articles
    ref over View, Controller, Repo: Sequence List All Articles (3.11.1)
end

opt Get Article Details
    ref over View, Controller, Repo: Sequence Get Article Details (3.11.2)
end

opt Create Article
    ref over View, Controller, Repo: Sequence Create Article (3.11.3)
end

opt Update Article
    ref over View, Controller, Repo: Sequence Update Article (3.11.4)
end

opt Update Article Relations
    ref over View, Controller, Repo: Sequence Update Article Relations (3.11.5)
end

opt Delete Article
    ref over View, Controller, Repo: Sequence Delete Article (3.11.6)
end
@enduml
```

---

## 3. Activities Flow (Swimlanes)

```plantuml
@startuml
|Admin|
start
:(1) Access Article Management;
:(2) View Article List;

|ArticleController|
:(3) Retrieve Article List;
note right: BR-01

|ArticleRepository|
:(4) Query All Articles;

|Admin|
:(5) Display Article List;
:(6) Choose Function;

if (Which Function?) then (List All)
    :(7.1) Go to List All Articles;
    note right: Ref: 3.11.1
    stop
elseif (Get Details) then
    :(7.2) Go to Get Article Details;
    note right: Ref: 3.11.2
    stop
elseif (Create) then
    :(7.3) Go to Create Article;
    note right: Ref: 3.11.3
    stop
elseif (Update) then
    :(7.4) Go to Update Article;
    note right: Ref: 3.11.4
    stop
elseif (Relations) then
    :(7.5) Go to Update Article Relations;
    note right: Ref: 3.11.5
    stop
else (Delete)
    :(7.6) Go to Delete Article;
    note right: Ref: 3.11.6
    stop
endif
@enduml
```

## 4. Business Rules

| Activity | BR Code   | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| :------- | :-------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **(5)**  | **BR-01** | **Displaying Rules:**<br>❖ The system renders the “ArticleManagementPage” screen by calling `ArticleController.listAll()`. (Refer to “Article Management” view in “View Description” file).<br>❖ It displays a datagrid containing columns for [ID], [Title], [Type], [Author], and [CreatedAt], populated with the list of articles.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **(6)**  | **BR-02** | **Choosing Rules:**<br>❖ The Admin selects a function from the sidebar or action menu. The system handles navigation via `Frontend_Navigation()`.<br>❖ If “View List” is selected, the system redirects to `3.11.1`.<br>❖ If “View Details” is selected, it redirects to the detail view `3.11.2`.<br>❖ If “Create Article” is selected, the system opens the creation form `3.11.3`.<br>❖ If “Update Article” is selected, it redirects to the edit interface `3.11.4`.<br>❖ If “Relations” is selected, it opens the relationship manager `3.11.5`.<br>❖ If “Delete Article” is selected, the system initiates the deletion process `3.11.6`.<br>❖ The Admin can only choose one feature at a time.                                                                                                                                                                 |
