# Use Case 3.10.0: Manage Location

## 1. Use Case Description

| Field              | Content                                                                                                                                  |
| :----------------- | :--------------------------------------------------------------------------------------------------------------------------------------- |
| **Name**           | Manage Location                                                                                                                          |
| **Description**    | This use case allows the **User** to choose one of the corresponding functions of CRUD to adjust **Location** information in the system. |
| **Actor**          | **All (Public)**                                                                                                                         |
| **Trigger**        | When the **User** clicks on the "**Location**" button on the navigation sidebar or accesses location-related features.                   |
| **Pre-condition**  | 1. **User**'s device must be connected to the internet.                                                                                  |
| **Post-condition** | The **Location** information is updated to the corresponding function that the **User** executes.                                        |

---

## 2. Sequence Flow

```plantuml
@startuml
autonumber
skinparam sequenceMessageAlign center

actor "User" as User
boundary "LocationPage" as View
control "LocationController" as Controller
entity "LocationRepository" as Repo

User -> View: Access Location page
View -> Controller: GET /locations
activate Controller
Controller -> Repo: findAll()
activate Repo
Repo --> Controller: LocationList
deactivate Repo
Controller --> View: Return list data
deactivate Controller
View --> User: Display list of Locations

User -> View: Choose function

opt Get All Locations
    ref over View, Controller, Repo: Sequence Get All Locations (3.10.1)
end
@enduml
```

---

## 3. Activities Flow (Swimlanes)

```plantuml
@startuml
|User|
start
:(1) Access Location Page;
:(2) View Location List;

|LocationController|
:(3) Retrieve Location List;
note right: BR-01

|LocationRepository|
:(4) Query All Locations;

|User|
:(5) Display Location List;
:(6) Choose Function;

if (Which Function?) then (Get All Locations)
    :(7.1) Go to Get All Locations;
    note right: Ref: 3.10.1
    stop
endif
@enduml
```

## 4. Business Rules

| Activity | BR Code   | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| :------- | :-------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **(5)**  | **BR-01** | **Displaying Rules:**<br>❖ The system loads the “LocationPage” screen via `Display_View()`. (Refer to “LocationPage” view in “View Description” file).<br>❖ The system renders a hierarchical tree or list view of the available locations (Provinces, Districts, Wards).                                                                                                                                                                                    |
| **(6)**  | **BR-02** | **Choosing Rules:**<br>❖ The user selects a function from the available options. The system uses `Frontend_Navigation()` to direct the user.<br>❖ If the user selects "View All Locations", the system redirects to the full location listing workflow (Ref: `3.10.1`).<br>❖ The User can only choose one feature at a time.                                                                                                                                 |
