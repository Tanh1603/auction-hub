# Use Case 3.8.0: Manage Auction Cost

## 1. Use Case Description

| Field              | Content                                                                                                                                       |
| :----------------- | :-------------------------------------------------------------------------------------------------------------------------------------------- |
| **Name**           | Manage Auction Cost                                                                                                                           |
| **Description**    | This use case allows the **Admin** to choose one of the corresponding functions of CRUD to adjust **Auction Cost** information in the system. |
| **Actor**          | **Admin**, **Auctioneer**                                                                                                                     |
| **Trigger**        | When the **Admin** clicks on the "**Auction Cost**" button on the navigation sidebar.                                                         |
| **Pre-condition**  | 1. **Admin**'s device must be connected to the internet.<br>2. **Admin** is signed in with their account.                                     |
| **Post-condition** | The **Auction Cost** information is updated to the corresponding function that the **Admin** executes.                                        |

---

## 2. Sequence Flow

```plantuml
@startuml
autonumber
skinparam sequenceMessageAlign center

actor "Admin" as User
boundary "AuctionCostPage" as View
control "AuctionCostController" as Controller
entity "AuctionCostRepository" as Repo

User -> View: Access Auction Cost Management
View -> Controller: GET /auction-costs
activate Controller
Controller -> Repo: findAll()
activate Repo
Repo --> Controller: AuctionCostList
deactivate Repo
Controller --> View: Return list data
deactivate Controller
View --> User: Display list of Auction Costs

User -> View: Choose function

' FIX: Use 'alt' with 'else' for mutually exclusive menu choices
' This renders as one clean block with separators instead of 5 stacked boxes
alt Create/Update Auction Costs
    ref over View, Controller, Repo: Sequence Create Update Auction Costs (3.8.1)
else Get Auction Costs
    ref over View, Controller, Repo: Sequence Get Auction Costs (3.8.2)
else Update Specific Cost Fields
    ref over View, Controller, Repo: Sequence Update Specific Cost Fields (3.8.3)
else Delete Auction Costs
    ref over View, Controller, Repo: Sequence Delete Auction Costs (3.8.4)
else Add Other Cost Item
    ref over View, Controller, Repo: Sequence Add Other Cost Item (3.8.5)
end

@enduml
```

## 3. Activities Flow (Swimlanes)

```plantuml
@startuml
|Admin|
start
:(1) Access Auction Cost Management;
:(2) View Auction Cost List;

|AuctionCostController|
:(3) Retrieve Auction Cost List;
note right: BR-01

|AuctionCostRepository|
:(4) Query All Auction Costs;

|Admin|
:(5) Display Auction Cost List;
:(6) Choose Function;

if (Which Function?) then (Create/Update)
    :(7.1) Go to Create/Update Auction Costs;
    note right: Ref: 3.8.1
    stop
elseif (Get Costs) then
    :(7.2) Go to Get Auction Costs;
    note right: Ref: 3.8.2
    stop
elseif (Update Fields) then
    :(7.3) Go to Update Specific Cost Fields;
    note right: Ref: 3.8.3
    stop
elseif (Delete) then
    :(7.4) Go to Delete Auction Costs;
    note right: Ref: 3.8.4
    stop
else (Add Other Cost)
    :(7.5) Go to Add Other Cost Item;
    note right: Ref: 3.8.5
    stop
endif
@enduml
```
